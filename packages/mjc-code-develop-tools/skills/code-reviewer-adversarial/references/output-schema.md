# Breaker 出力スキーマ（反例中心 7 フィールド）

Breaker が生成する全指摘は、以下の7フィールドを必ず埋める。反証可能性を強制するためのスキーマであり、フィールド欠損は許容しない（書けないフィールドは「N/A — <理由>」で埋める）。

## フィールド定義

| フィールド | 必須 | 内容 |
|---|---|---|
| 指摘内容 | ✓ | 1文で「何が壊れているか」を記述。原因や修正には触れない |
| 再現条件 | ✓ | どの入力・状態・呼び出し順序で再現するか。環境前提（OS・runtime・依存バージョン）を含める |
| 最小反例 | ✓ | 実行可能なテストコード（テスト生成モード）または最小の入力・手順（記述のみモード）。可能な限り単体テストとして貼る |
| 影響範囲 | ✓ | 影響するコンポーネント・ユーザー・データ。「単一リクエストのみ」か「システム全体」かを区別する |
| 重大度 | ✓ | Critical / High / Medium / Low（[severity-rubric.md](severity-rubric.md)） |
| 修正案 | ✓ | 方向性のみ。パッチレベルの書き直しは行わない |
| 確信度 | ✓ | High / Medium / Low（[severity-rubric.md](severity-rubric.md)）。反証不能な推測は Low |

## レコード書式

```markdown
#### 指摘 <連番>: <1行タイトル>

- **指摘内容**: <1文>
- **再現条件**: <入力・状態・順序・環境>
- **最小反例**:
  ```<lang>
  <テストコード または 最小入力/手順>
  ```
  実行結果: <failed / passed / not-run>
- **影響範囲**: <範囲>
- **重大度**: Critical | High | Medium | Low
- **修正案**: <方向性>
- **確信度**: High | Medium | Low
- **タグ**: [UNVERIFIED] <記述のみで反証していない場合のみ付与>
```

## 書き方の注意

- 「〜かもしれない」「〜の可能性がある」で書かれた指摘は確信度 Low
- 推測の根拠が diff 以外にある場合（外部仕様・ライブラリ挙動）はコメントで参照先を明記
- 同じ根本原因の複数症状は1指摘に統合する（Judge が判断しやすくするため）
- 重複指摘は persona 横断でマージし、タイトルに `(Security+Performance)` 等を注記

## テスト実行モードでの扱い

- `failed` → 「検証済み反例」として採用、そのまま Breaker 報告書に残す
- `passed` → 仮説が外れた。Breaker 内で破棄（Judge に送らない）
- `not-run` → テスト記述のみモード。`[UNVERIFIED]` タグ必須、確信度は High にはしない
