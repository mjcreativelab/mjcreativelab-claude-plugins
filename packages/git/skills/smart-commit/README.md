# smart-commit

現在の git 差分を作業内容ごとに適切な単位で分割し、日本語 conventional commits でコミットするスキル。

## 使い方

```
/smart-commit
/smart-commit -p chore フォルダの差分は省く
/smart-commit -p e2e の変更だけコミットして
```

または「コミットして」「差分をコミット」と伝える。

## オプション

| オプション        | 説明                                                 |
| ----------------- | ---------------------------------------------------- |
| `-p <プロンプト>` | コミット対象の選別やメッセージに関する追加指示を渡す |

## ブランチ連動フィルタリング

`main` 以外のブランチにいる場合、ブランチ名（と紐づく Issue）からそのブランチの目的を推定し、目的に合致する差分のみをコミット対象にする。合致しない差分は working tree に残る。

`-p` オプションの指示はブランチ連動フィルタリングより優先される。

## 推奨: 新規セッションで実行する

このスキルは **作業セッションとは別の新規セッションで実行する** ことを推奨する。

- 作業セッションにはコード探索・編集・試行錯誤の履歴が大量に蓄積されており、スキル実行時にそのコンテキストがすべてトークンとして消費される
- 新規セッションなら、スキルが必要な情報（git diff, git status 等）だけを読み込むため、トークン効率が圧倒的に良い
- **作業が一段落したら、新規セッションで `/smart-commit` を実行するのがベスト**
